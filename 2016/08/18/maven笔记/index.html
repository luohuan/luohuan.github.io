maven学习笔记--何为maven

  • 新建,模板,小书匠

何为maven

maven解释为知识的积累，是一个跨平台的项目管理工具。
maven主要服务于基于Java平台的项目构建、依赖管理和项目信息管理

maven 解决的核心问题是简化程序员的工作，让单元测试、打包、部署等只需要一条命令能够自动的执行。构建服务。
maven不仅仅是构建工具还是一个依赖管理工具和项目信息管理工具

maven使用入门

maven项目的核心是pom.xml (project object model) 项目对象模型
pom定义了项目的基本信息，用于描述项目如何构建，声明项目依赖项。

project 是所有po.xml的根元素，还声明了一些pom相关的命名空间及xsd文件
元素不是必须，但是能让第三方文件快速的编辑pom文件

单元测试小知识

一个典型的单元测试包含三个部分

1. 准备测试类及数据
2. 执行要测试的行为
3. 检查结果

所有需要执行的测试方法都以test开头同时需要有@test注解

maven的主要命令

  1. mvn clean compile 编译
  2. mvn clean test 测试
  3. mvn clean package 打包
  4. mvn clean install 安装
    如果直接执行后面的命令会依次的执行之前的命令。
    mvn clean 清除目标目录生成的结果

maven坐标

maven定义的规则： 世界上任何一个构件都可以使用maven坐标唯一的标识，maven坐标的元素包括groupId、artifactId 、version、 packaging、classifier。
只要我们提供正确的坐标元素，maven就能找到对应的构件

maven坐标为各种构件引入了秩序，任何一个构件都必须明确定义自己的坐标

  1. groupId : 定义当前maven 项目隶属的实际项目。一个实际项目往往会划分出很多的模块，groupId不应该只定义到组织级别。通常是公司名称加项目名称
  2. artifactId : 该元素定义实际项目中的一个maven项目也就是一个模块。推荐的做法是使用实际项目作为前缀，默认情况下Maven生成的构建会以artifactId-version作为名称
  3. version 该元素定义maven项目当前所处的版本
  4. packaging :打包的方式 默认是使用jar
  5. classifier: 该元素用来帮助定义构建输出的一些附属构建。 不能直接定义
